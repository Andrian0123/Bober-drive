# 📊 Bober-Drive Dashboard v3.0.1

Веб-интерфейс для **real-time мониторинга** статистики и производительности Bober-Drive демона.

## 🚀 Быстрый старт

### 1️⃣ Запуск (2 способа)

**Вариант A: Быстрый запуск (рекомендуется)**
```bash
python launch_dashboard.py
```
✅ Автоматически:

- Запустит демон для `./docs`

```bash
    --port 8000 \
    --host 127.0.0.1
```

### 2️⃣ Откройте браузер
```
http://localhost:8000
```

## 📈 Что показывает дашборд

### Основные метрики (в реальном времени)

| Метрика | Описание | Единица |
|---------|---------|---------|
| **Files Scanned** | Всего файлов обнаружено при сканировании | шт |
| **Files Indexed** | Файлов успешно добавлено в индекс | шт |
| **Searches** | Количество выполненных поисков | шт |
| **Cache Hit Rate** | Процент успешных попаданий в кеш | % |
| **Cache Size** | Текущий размер кеша в памяти | MB |
| **Avg Search Time** | Среднее время выполнения поиска | ms |
| **Startup Time** | Время инициализации демона при старте | ms |
| **Reindex Queue** | Файлы, ожидающие переиндексирования | шт |

### Графики

#### 1. **Indexing Progress** (Прогресс индексирования)
- Doughnut chart
- Показывает: сколько файлов индексировано vs осталось
- Обновляется в реальном времени при добавлении файлов

#### 2. **Cache Performance** (Производительность кеша)
- Bar chart
- Показывает: попадания в кеш vs промахи
- Помогает оценить эффективность кеширования

### Live Status (Живая лента событий)
- Последние 50 событий в хронологическом порядке (новые сверху)
- Цвет кодирование: INFO (синий), WARN (оранжевый), ERROR (красный)

### Индикаторы статуса

| Статус | Значение |
|--------|----------|
| 🟢 **Ready** | Демон активен, слушает события |
| 🟠 **Initializing** | Фаза инициализации (сканирование проекта) |
| 🔴 **Stopped** | Демон остановлен |
| 🔴 **Disconnected** | Потеря соединения WebSocket |

## 🔧 Как работает

### Архитектура

```
┌─────────────────────────────────────────┐
│   Браузер (HTML5 + Chart.js)           │
│   (Real-time Dashboard UI)              │
└────────────┬────────────────────────────┘
             │ WebSocket (http://localhost:8000/ws/metrics)
             │ Обновление каждые 1 сек
             ↓
┌─────────────────────────────────────────┐
│   FastAPI Server (dashboard.py)         │
│   - GET /              (HTML страница)  │
│   - GET /api/status    (JSON метрики)   │
│   - WebSocket /ws/metrics (real-time)   │
└────────────┬────────────────────────────┘
             │
             ↓
┌─────────────────────────────────────────┐
│   NexusAutonomousDaemon (в памяти)     │
│   - get_status()      → состояние      │
│   - get_metrics()     → метрики        │
│   - get_cache_stats() → кеш стат      │
└─────────────────────────────────────────┘
```

### Поток данных

1. **Frontend** (браузер): открывает WebSocket соединение
2. **Backend** (FastAPI): каждую секунду отправляет свежие метрики
3. **Daemon** (Python): предоставляет API для получения статистики
4. **Frontend**: обновляет графики и счётчики в реальном времени

## 🎨 Интерфейс

### Цветовая схема (Gradient Purple)
- **Фон**: линейный градиент от #667eea до #764ba2 (фиолетово-голубой)
- **Карточки**: белые (#fff) с цветными левыми бордерами
- **Графики**: встроенные в Chart.js (голубой/зелёный)

### Адаптивность
- 📱 Полностью responsive (мобильные, планшеты, десктоп)
- 📱 На мобильных: сетка 1-2 колонны вместо 4-х
- 🖥️ На десктопе: оптимальная сетка 4 колонны

### Производительность
- **Размер HTML**: ~15 KB (встроенный, без внешних файлов)
- **JS зависимости**: только Chart.js (CDN, ~200 KB)
- **Память**: <5 MB в браузере
- **Обновления**: debounced каждые 1 сек

## 🛠️ API Endpoints

Если нужно интегрировать метрики в свой код:

### GET /api/status
Получить текущий статус в виде JSON

```python
import requests

response = requests.get('http://localhost:8000/api/status')
status = response.json()

print(f"Files indexed: {status['files_indexed']}")
print(f"Cache hit rate: {status['cache_hit_rate']}%")
```

Ответ:
```json
{
  "state": "READY",
  "uptime": 145.3,
  "files_scanned": 195,
  "files_indexed": 195,
  "cache_hit_rate": 84.7,
  "cache_size_mb": 245.3,
  "search_queries": 12,
  "avg_search_time_ms": 2.1,
  "startup_time_ms": 8234,
  "timestamp": "2026-07-21T11:30:00.123456"
}
```

### WebSocket /ws/metrics
Подписаться на real-time метрики

```javascript
const ws = new WebSocket('ws://localhost:8000/ws/metrics');

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log(`Indexed: ${data.files_indexed}/${data.files_scanned}`);
};
```

## 🚨 Troubleshooting

### Проблема: "Ошибка подключения WebSocket"
**Причина:** сервер не запущен или неправильный адрес

**Решение:**
```bash
# Проверьте, запущен ли server
curl http://localhost:8000/api/status

# Перезагрузите браузер (Ctrl+Shift+R)
```

### Проблема: "Демон не запустился"
**Причина:** неправильный путь к документации

**Решение:**
```bash
python driver/dashboard.py --project /absolute/path/to/docs
```

### Проблема: "Высокое использование памяти"
**Причина:** большой кеш или много файлов

**Решение:** ограничьте размер кеша в конфигурации демона:
```python
config = DaemonConfig(
    file_cache_size_mb=256,  # Вместо 500
    file_cache_max_entries=500  # Вместо 1000
)
```

## 📊 Интерпретация метрик

### Здоровые значения

| Метрика | Нормально | Забота |
|---------|-----------|--------|
| **Cache Hit Rate** | > 80% | < 50% — кеш не эффективен |
| **Avg Search Time** | 1-5 ms | > 100 ms — медленный поиск |
| **Startup Time** | < 15 сек | > 60 сек — долгая инициализация |
| **Reindex Queue** | 0 (в состоянии READY) | > 10 — файлы копятся |

### Что означают события в Live Status

- `[INFO] Scanned 195 files` → сканирование завершено
- `[INFO] Indexed X files` → индексирование завершено
- `[WARN] Cache read failed` → файл из кеша не удалось прочитать
- `[ERROR] Scan failed: ...` → ошибка при сканировании проекта

⚠️ **По умолчанию:**
- Дашборд слушает только на `127.0.0.1` (локальная машина)
- Нет аутентификации (добавьте, если открываете в сеть)
- Не передаёт данные во внешние сервисы

**Для боевого окружения:**
```bash
# 1. Используйте reverse proxy (nginx)
# 2. Добавьте HTTPS
# 3. Включите аутентификацию
# 4. Ограничьте доступ по IP
```
